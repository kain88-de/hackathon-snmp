**Goal:** Build the OIDviz browser app — a Vue 3 + TypeScript viewer for
`.oidtrace.jsonl.gz` files that lets a monitoring admin answer "is this device
behaving correctly?" with three interactive views.

**Architecture:** A single-page app with a two-column layout (fixed 220px
sidebar + flex main). All trace parsing happens off the main thread in a Web
Worker. Pure computation modules (filter logic, incident clustering, OID trie)
are plain TypeScript with no Vue dependencies, so they are testable with
`bun test` without a browser. Vue 3 Composition API drives reactivity in
components.

**Tech Stack:** Vue 3 · TypeScript · Bun · Vite · `@vitejs/plugin-vue` · Biome
(formatting + linting, all rule groups enabled) · Oxlint (all rules enabled via
`-D all`, with `.oxlintrc.json` to suppress false positives) · ESLint +
`eslint-plugin-vue` (Vue template conventions, `.vue` files only,
`--max-warnings 0`) · vue-tsc (template-aware type checking) · Playwright
(`@playwright/test`, browser-based e2e smoke tests) · axe-core CLI (rendered
a11y gate) · see `docs/dev-guidelines/web-guardrails.md` for the full toolchain
and feedback loop.

### Generated types (`src/lib/types.gen.ts`)

Run once at setup and again after schema changes:

```sh
bunx json-schema-to-typescript ../docs/trace-format.schema.json -o src/lib/types.gen.ts
```

This produces TypeScript interfaces for the five top-level record types:
`Header`, `SystemInfo`, `Exchange`, `Event`, `Summary`. Do not hand-write these.

**Trap — nested type names are not guaranteed.** `Attempt`, `Request`,
`Response`, `Settings`, and `Varbind` are inline object schemas inside
`exchange` and `header`, not top-level `$defs`. `json-schema-to-typescript`
synthesises names for them (often something like `Exchange1`, `Header1`, or just
inlines them). **Run `just gen-types` in task 1 and inspect the actual output
before writing `model.ts` or `ParseResult`.** The plan uses
`Attempt`/`Request`/`Settings` etc. as readable shorthand; pin the real
generated names before implementing.

**`received_at` nullability**: The generated type for the attempt's
`received_at` field may be `number | null`. Verify the generated output
preserves this — it must be nullable so timeout detection works without type
casts downstream.

### Type contracts (binding — declare before implementing)

These shapes are law. Where prose elsewhere disagrees, **this section wins**.
Generated names from `types.gen.ts` are confirmed real exports — import
`Header`, `SystemInfo`, `Exchange`, `Event`, `Summary`, `Varbind`, `Oid`
directly.

#### Type-driven development

**Encode correctness in the type system. If a constraint can be a compile
error, it must be.** Branded types for domain quantities, discriminated unions
with exhaustiveness for state machines, runtime validation before external data
enters the typed domain, `readonly` inputs to pure functions.

Binding constraints: `enum` not `const enum` (Biome bans it); `T | null` not
`?` for present-but-empty (`exactOptionalPropertyTypes`); no `any` on public
surfaces.

#### Contracts by module

**`model.ts`** — no logic except `asOid`. Only `interface`, `type`, `enum`, and
that one constructor.

```ts
export type OidString = string & { readonly _brand: "OidString" };
export function asOid(s: string): OidString; // sole sanctioned cast — worker only

export interface DomainExchange {
  seq: number;
  rtt: number; // ms
  isTimeout: boolean;
  violations: string[]; // open enum per format spec — string[] is correct
  attemptCount: number;
  requestOid: OidString;
  responseOids: OidString[]; // unique, 7-arc-truncated; [] when no response
  sentAtMs: number;
  receivedAtMs: number;
}

export interface ParseResult {
  exchanges: DomainExchange[];
  header: Header;
  summary: Summary | null; // null not absent — exactOptionalPropertyTypes
  systemInfo: SystemInfo | null;
  parseMs: number;
  truncated: boolean;
}

// Facet axes: Performance is mutually exclusive (exactly one state per exchange);
// Correctness is independent; retryOnly is an orthogonal attribute.
export interface FacetState {
  perf: "any" | "fast" | "slow" | "timeout"; // default 'any'
  corr: "any" | "violations"; // default 'any'
  retryOnly: boolean; // default false
  slowMs: number; // default 1000ms; UI input is seconds
}

export interface Incident {
  startIdx: number;
  endIdx: number;
  startSeq: number;
  endSeq: number;
  members: number[]; // indices into the exchanges array
  peakRtt: number; // ms
  retryCount: number; // sum of (attemptCount - 1) over members
  timeoutCount: number;
  violationTypes: Set<string>; // Set encodes "distinct" — score uses .size
  region: string; // a label, NOT OidString
  score: number;
}

// Retire Severity enum. Replace severity: Severity with flags: TrieNodeFlags.
// Math.max(node.severity, …) as Severity casts are the exact pattern this
// section exists to prevent — they bypass the type system to coerce a number
// back to an enum. flags needs no casts.
export interface TrieNodeFlags {
  slow: boolean;
  violation: boolean;
  retry: boolean; // timeout absent by construction: no response OIDs
}

export interface TrieNode {
  arc: string; // arc label — plain string, not OidString
  fullOid: OidString;
  name: string | null;
  children: Map<string, TrieNode>;
  leaves: TrieLeaf[];
  expanded: boolean; // intentionally mutable — only mutable domain field
  stats: { count: number; maxRtt: number; violationCount: number };
  flags: TrieNodeFlags; // OR-rolled from children + own leaves
}

// oid is the response OID this placement was filed under — differs from
// exchange.requestOid in GETNEXT/BULK walks (inconsistency #12 fix).
export interface TrieLeaf {
  exchange: DomainExchange;
  oid: OidString; // placement path (response OID), not requestOid
  shared: boolean;
}

export type FlatRow =
  | { kind: "node"; depth: number; node: TrieNode }
  | {
      kind: "leaf";
      depth: number;
      exchange: DomainExchange;
      oid: OidString;
      shared: boolean;
    };

export interface WorkerRequest {
  type: "parse";
  buffer: ArrayBuffer;
}
export type WorkerResponse =
  | { type: "result"; data: ParseResult }
  | { type: "error"; message: string };

export type AppState =
  | { phase: "landing" }
  | { phase: "loading" }
  | { phase: "viewer"; result: ParseResult }
  | { phase: "error"; message: string };

// 'findings'='Findings', 'incidents'='Incident Stack', 'minimap'='Minimap + Detail', 'oidtree'='OID Tree'
export type ActiveView = "findings" | "incidents" | "minimap" | "oidtree";
```

### App.vue

State: `facetState` init
`{ perf: 'any', corr: 'any', retryOnly: false, slowMs: 1000 }`, `activeView`
init `'findings'`. Computed: `filteredExchanges` (facet-filtered, shared by all
views); `incidents` built from the **full** list (clustering is anomaly-driven,
not facet-driven); `oidRoot = buildTrie(filteredExchanges)` → `rollup` →
`autoExpand` → `flatten`. Full trie rebuild on facet change; expand state not
preserved. `result.truncated` → sidebar warning with `aria-live="polite"`.

### Components — non-obvious traps only

Shapes and props/emits are in §Type contracts. The spec is the authoritative
source for structure and UX. Only traps below.

**Sidebar**: radio groups need `<fieldset>`/`<legend>` for a11y (not just
labels). The file section contains only the "Open file" button and hidden file
input — no fixture shortcut buttons.

**LandingScreen**: `role="alert"` on the error message (not a dialog). Drop zone
is a `<div role="region" tabindex="0">`, not a `<button>` — `Enter`/`Space` must
open the hidden file input via JS.

**FindingsByCategory**: sections flatten into one virtualised list with
`kind: 'header' | 'row'` rows for the spacer calculation — same pattern as OID
Tree. Fast section only rendered when non-empty. **Section headers are
collapsible fold-outs**: each header is a `<button aria-expanded>` with a chevron
(▼ open / ▶ closed); clicking toggles the section. All three sections start
expanded. When collapsed, that section's rows are excluded from the flat list —
not hidden with CSS.

**IncidentStack**: chip colours use `--dim-*` tokens (`--dim-timeout`,
`--dim-violation`, `--dim-slow`, `--dim-retry`), **not** the retired
`severity-err/warn/info` classes. Modal exchange table shows only
`matchesFacets`-passing members.

**IncidentModal**: on open, move focus to `<h2>`; on close, return focus to the
trigger row. `role="dialog"` on the panel, not the overlay.

**MinimapDetail**: canvas reads `--dim-*` colours via `getComputedStyle` at draw
time. Layout constants: `MINI_H=80`, `ML=64`, `MT=24`, `RH=9`, `RG=3`, `BH=6`,
`MAX_H=32_767`. Display precedence for single-pixel colour:
`Timeout > Violation > Slow > Retry`. Timestamps are relative to walk start
(`sentAtMs` already offset in the worker). Detail canvas height:
`MT + rows × (RH + RG) + 20` capped at `MAX_H`. `autoFocus()` on data load:
densest/worst bucket scored `(viol ? 1e9 : 0) + maxRtt × 1000 + count`.

**OidTree**: `TrieNode.expanded` is mutated in place; the component emits
`reflatten` and the parent re-flattens. After collapse-all, the flat list
shrinks — the virtualiser must not hold stale row indices. Leaf rows display
`exchange.requestOid` (the full polled OID, e.g. `1.3.6.1.2.1.1.1.0`), **not**
`row.oid`. The placement OID (`row.oid`) is 7-arc-truncated for trie placement —
every sibling leaf under the same parent node shares the same truncated OID, so
using it as a display label produces a flat list of duplicates instead of a
meaningful tree leaf.

### CSS (`app.css`)

`:root` defines `--dim-slow`, `--dim-violation`, `--dim-retry`, `--dim-timeout`,
`--dim-none` and their `-bg` variants. No dark mode overrides — light mode only.
DOM components use only `var(--…)` — no hard-coded hex. Canvas reads `--dim-*`
via `getComputedStyle`. `.sr-only` for screen-reader-only text.

The tokens must cover: background, surface, border, primary colour, ok/warn/err
semantic colours, text, muted text, sidebar background, sidebar text, sidebar
muted, sidebar border, monospace font stack.

Exact colour values: derive from the prototype's `:root` and existing style
rules. The prototype uses hard-coded hex; move these to custom properties and
ensure ≥ 4.5:1 contrast ratio for text/background pairs.

DOM-based content (sidebar, incident modal, OID Tree rows) must use only
`var(--...)` tokens — no hard-coded colours. Canvas-based views
(`MinimapDetail.vue`) read `--dim-*` colour tokens via `getComputedStyle` at
draw time.

**Viewport constraints**: The sidebar is `220px` fixed width; the main area uses
`flex: 1` with no `max-width` cap (stretches to fill all available width on 4K
and ultrawide). No responsive breakpoints below `1280px` — the layout is not
tested below this width.

## Vue 3 traps

- **Canvas refs**: `ref<HTMLCanvasElement | null>(null)` — the ref is `null`
  until mounted. All `getContext()` and `ResizeObserver` calls must be in
  `onMounted`.
- **Non-reactive trie**: do **not** wrap `TrieNode` in `reactive()` or `ref()` —
  deep reactivity on 100k leaves kills performance. `oidRoot` is replaced on
  facet change (triggers `computed`); `expanded` mutations are handled by the
  explicit reflatten emit.
- **`defineEmits`** not callback props — Vue's `@event` convention throughout.
- **`v-for` keys**: `seq` for exchange rows, `fullOid` for trie nodes.
- **`responseOids` source**: use `exchange.response?.varbinds`, **not**
  `attempts[].response.varbinds` — the prototype has a historical artefact on
  the per-attempt path.
