# OIDviz Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **BEFORE ANYTHING ELSE:** Read `oidviz/docs/web-guardrails.md`. It is a hard constraint list — wrong package manager,
> wrong linter, or wrong tsc invocation is a task failure. No exceptions.

**Goal:** Build the OIDviz browser app — a Vue 3 + TypeScript viewer for `.oidtrace.jsonl.gz` files that lets a
monitoring admin answer "is this device behaving correctly?" with three interactive views.

**Architecture:** A single-page app with a two-column layout (fixed 220px sidebar + flex main). All trace parsing
happens off the main thread in a Web Worker. Pure computation modules (filter logic, incident clustering, OID trie) are
plain TypeScript with no Vue dependencies, so they are testable with `bun test` without a browser. Vue 3 Composition API
drives reactivity in components.

**Tech Stack:** Vue 3 · TypeScript · Bun · Vite · `@vitejs/plugin-vue` · Biome (formatting + linting, all rule groups
enabled) · Oxlint (all rules enabled via `-D all`, with `.oxlintrc.json` to suppress false positives) · axe-core CLI
(rendered a11y gate) · see `oidviz/docs/web-guardrails.md` for the full toolchain and feedback loop.

### Generated types (`src/lib/types.gen.ts`)

Run once at setup and again after schema changes:

```sh
bunx json-schema-to-typescript ../docs/trace-format.schema.json -o src/lib/types.gen.ts
```

This produces TypeScript interfaces for the five top-level record types: `Header`, `SystemInfo`, `Exchange`, `Event`,
`Summary`. Do not hand-write these.

**Trap — nested type names are not guaranteed.** `Attempt`, `Request`, `Response`, `Settings`, and `Varbind` are inline
object schemas inside `exchange` and `header`, not top-level `$defs`. `json-schema-to-typescript` synthesises names for
them (often something like `Exchange1`, `Header1`, or just inlines them). **Run `just gen-types` in task 1 and inspect
the actual output before writing `model.ts` or `ParseResult`.** The plan uses `Attempt`/`Request`/`Settings` etc. as
readable shorthand; pin the real generated names before implementing.

**`received_at` nullability**: The generated type for the attempt's `received_at` field may be `number | null`. Verify
the generated output preserves this — it must be nullable so timeout detection works without type casts downstream.

### Type contracts (binding — declare before implementing)

Every type below is a contract. Implementers work to these shapes; they do not discover them mid-task. The discipline mirrors the OIDtrace plan: structural vocabularies are enums, structured identifiers are branded types, and module boundaries carry exact signatures. Where the prose elsewhere in this plan disagrees with a shape here, **this section wins**.

The generated names in `types.gen.ts` are confirmed real top-level exports — `Header`, `SystemInfo`, `Exchange`, `Event`, `Summary`, `Varbind`, and `Oid` (a `type Oid = string` alias). The synthesised-name trap from the "Generated types" note did **not** materialise; import these names directly. `Oid` from `types.gen.ts` is plain `string` (schema-level); re-brand at the parse boundary (see `OidString` below).

#### Type-driven rules (TypeScript)

Strict by default. A bare `string`/`number` may cross a module boundary only when the domain is genuinely open (e.g. a violation string is an open enum per the format spec; an RTT in ms is an open numeric range). Everything else is closed.

1. **Branded types for structured identifiers.** A dotted OID is not an arbitrary string — it is `OidString`. Region labels, well-known names, and raw arc components are plain `string` and must not flow into an `OidString` slot. The brand is erased at runtime (zero cost); it exists only to stop transpositions at compile time. Re-brand exactly once, at the parser boundary, via `asOid()` — the single sanctioned cast in the codebase.

2. **`enum`, never `const enum`.** Biome bans `const enum` as bundler-incompatible. Use plain `enum`. *(Note: `Severity` is retired — see `TrieNodeFlags`. No numeric-coded enum survives in the domain model.)*

3. **String-literal unions for closed vocabularies that never need a numeric value.** `perf`, `corr`, `phase`, `kind`, `activeView`, `WorkerRequest.type` are literal unions, not enums — they are used purely for discrimination.

4. **Discriminated unions for state and messages.** `AppState`, `WorkerResponse`, and `FlatRow` carry a literal discriminant (`phase` / `type` / `kind`) as the first field. A `switch` over the discriminant with no `default` arm gives compiler-checked exhaustiveness. Do not add a `default: throw` that defeats this.

5. **`readonly` arrays at pure-function inputs.** Pure modules (`filters.ts`, `incidentStack.ts`, `oidTrie.ts`) take `readonly DomainExchange[]` — project style, **not** `ReadonlyArray<…>`. The one intentional exception is `TrieNode.expanded`, mutated in place by the OID Tree (documented at its declaration).

6. **No `any` in any public surface.** Worker `MessageEvent` data is typed `MessageEvent<WorkerResponse>` at the listener, not cast through `any`.

7. **`noUncheckedIndexedAccess` implications.** Every indexed access (`exchanges[i]`, `attempts[0]`, `members[0]`) is `T | undefined`. Two sanctioned patterns: (a) a guard (`if (ex === undefined) continue;`) where a hole is genuinely possible; (b) a non-null assertion (`as DomainExchange`) **only** where the format spec guarantees presence (`minItems: 1` on `attempts`, member indices sourced from the same array), each with a one-line comment citing the guarantee.

8. **`exactOptionalPropertyTypes` implications.** An optional field may be *absent* but may **not** be explicitly set to `undefined`. Use `T | null` for "computed-but-empty" values (`summary: Summary | null`, `name: string | null`, `received_at: number | null`). Reserve bare `?` for genuinely-may-be-omitted object keys.

### Domain model (`src/lib/model.ts`)

Defines all domain and inter-component types. No logic — only `interface` and `type` declarations.

**`DomainExchange`** — enriched form of a schema `Exchange` record with computed fields baked in at parse time:

| Field          | Type          | Source                                                                                |
| -------------- | ------------- | ------------------------------------------------------------------------------------- |
| `seq`          | `number`      | `exchange.seq`                                                                        |
| `rtt`          | `number` (ms) | Computed from attempts (see RTT rule below)                                           |
| `isTimeout`    | `boolean`     | `attempts.some(a => a.received_at === null)`                                          |
| `violations`   | `string[]`    | `exchange.violations ?? []`                                                           |
| `attemptCount` | `number`      | `exchange.attempts.length`                                                            |
| `requestOid`   | `OidString`   | `exchange.request.oids[0]`                                                            |
| `responseOids` | `OidString[]` | Unique 7-arc-truncated OIDs from `exchange.response?.varbinds` (empty if no response) |
| `sentAtMs`     | `number`      | `exchange.attempts[0].sent_at * 1000`                                                 |
| `receivedAtMs` | `number`      | RTT end time in ms (same formula as `rtt`)                                            |

**RTT rule**: Exchange RTT is `(last.received_at - first.sent_at) * 1000`. If the last attempt timed out
(`received_at === null`), use `(last.sent_at + timeout_s - first.sent_at) * 1000`. `timeout_s` comes from the trace
header (`header.settings.timeout_s`), not from the exchange itself. This must be resolved at parse time in the worker.

**`responseOids` truncation**: OID string → take first 7 arc components → rejoin with `.`. For `"1.3.6.1.2.1.2.2.1.1.1"`
→ `"1.3.6.1.2.1.2"`. Deduplicate across all varbinds in the exchange.

**`ParseResult`** — what the worker emits on success:

```
{
  exchanges: DomainExchange[]
  header: Header               // schema-generated
  summary: Summary | null      // absent if no summary record in file
  systemInfo: SystemInfo | null  // type: 'system_info', point: 'start'
  parseMs: number              // wall-clock ms for the full parse
  truncated: boolean           // true if file was cut mid-line
}
```

**`FacetState`** — reactive state held in `App.vue`, passed to all consumers:

```
{
  perf: 'any' | 'fast' | 'slow' | 'timeout'  // default 'any'
  corr: 'any' | 'violations'                  // default 'any'
  retryOnly: boolean                          // default false
  slowMs: number                              // default 1000ms; input is seconds, UI converts
}
```

An exchange **passes** all facets when:
- `perf === 'any'` or the exchange's performance state matches
- `corr === 'any'` or `violations.length > 0`
- `!retryOnly` or `attemptCount > 1`

Performance states are mutually exclusive: Fast = `!isTimeout && rtt ≤ slowMs`, Slow = `!isTimeout && rtt > slowMs`, Timeout = `isTimeout`.

**`Incident`** — output of `buildIncidents()`:

```
{
  startIdx: number    // index into the exchanges array (for walk-position bar)
  endIdx: number
  startSeq: number
  endSeq: number
  members: number[]   // indices into the exchanges array
  peakRtt: number     // ms
  retryCount: number  // total extra attempts (sum of attemptCount-1 over members)
  timeoutCount: number  // count of member exchanges with isTimeout=true
  violationTypes: Set<string>  // distinct violation strings across all members
  region: string      // dominant OID region name
  score: number       // for sorting (highest first)
}
```

**`TrieNode`** — internal node of the OID trie. All fields are effectively readonly after `rollup()` completes except
`expanded`, which is the only intentionally mutable field in the entire domain model (toggled by the OID Tree UI):

```
{
  arc: string          // this node's OID arc label
  fullOid: OidString   // full OID path up to this node
  name: string | null  // well-known name or null
  children: Map<string, TrieNode>
  leaves: TrieLeaf[]
  expanded: boolean    // intentionally mutable — OID Tree expand/collapse state
  stats: { count: number; maxRtt: number; violationCount: number }
  severity: Severity   // enum: Severity.Ok=0, Severity.Slow=1, Severity.Violation=2; propagated upward
}
```

**`TrieLeaf`**:

```
{
  exchange: DomainExchange
  shared: boolean  // true if this exchange appears under more than one trie node
}
```

**`FlatRow`** — flattened trie for virtualised rendering:

```
| { kind: 'node'; depth: number; node: TrieNode }
| { kind: 'leaf'; depth: number; exchange: DomainExchange; shared: boolean }
```

**`OidString`** — branded nominal type. `type OidString = string & { readonly _brand: 'OidString' }`. Exposes a single
constructor `asOid(s: string): OidString` (a cast, not validation — the worker is the only call site). All OID-carrying
fields in `DomainExchange`, `lookupOidName`, and the algorithm signatures use `OidString` rather than plain `string`.
This prevents region labels, display names, and raw arc components from being passed where a dotted OID is expected. The
generated types in `types.gen.ts` use plain `string` for OID fields — the worker casts to `OidString` at the single
parse boundary (same principle as the Python plan's "strings only at boundaries via `Oid.from_str` / `str(oid)`").

**`WorkerRequest`** and **`WorkerResponse`** — typed discriminated unions for the worker message protocol. Both are
exported from `model.ts` and imported by both `parser.worker.ts` and `App.vue`, so the compiler checks that both sides
handle the same set of cases:

```
interface WorkerRequest { type: 'parse'; buffer: ArrayBuffer }

type WorkerResponse =
  | { type: 'result'; data: ParseResult }
  | { type: 'error'; message: string }
```

The `switch (msg.type)` in `App.vue` gains exhaustiveness checking — an unhandled variant becomes a type error.

**`AppState`** — discriminated union for the top-level state machine:

```
| { phase: 'landing' }
| { phase: 'loading' }
| { phase: 'viewer'; result: ParseResult }
| { phase: 'error'; message: string }  // non-gzip, not for truncation
```

### Parser worker (`src/lib/parser.worker.ts`)

A standard Web Worker (not a module worker — use
`new Worker(new URL('./lib/parser.worker.ts', import.meta.url), { type: 'module' })` from `App.vue` so Vite bundles it).

**Message protocol**: both directions use the `WorkerRequest` and `WorkerResponse` discriminated unions from `model.ts`.
Import them in both the worker and `App.vue` so the compiler enforces shape at both ends. Cast incoming `MessageEvent`
data to `WorkerResponse` immediately on receipt; cast outgoing posts to `WorkerRequest` at the send site.

- Into the worker: `{ type: 'parse', buffer: ArrayBuffer }` — the raw file bytes, transferred (not copied).
- Out of the worker on success: `{ type: 'result', data: ParseResult }`.
- Out of the worker on non-gzip: `{ type: 'error', message: string }` — the app shows an inline error and returns to the
  landing screen.

**Parse algorithm**:

1. Wrap `buffer` in a `Blob`, open a `DecompressionStream('gzip')`, pipe and read chunks via
   `ReadableStreamDefaultReader`. If the decompression stream throws (not a valid gzip file), post `{ type: 'error' }`.
2. Decode chunks with `TextDecoder` (using `stream: true` to handle multi-byte code points split across chunks).
3. Buffer text and split on `\n`. For each complete line: `JSON.parse(line)`, then dispatch by `rec.type`.
4. When the reader is exhausted, check if the buffer has leftover bytes (truncated last line) — if so, set
   `truncated: true` in the result.
5. Record types to collect: `header` (first occurrence), `summary` (last occurrence wins), `system_info` with
   `point: 'start'`. All `exchange` records are enriched into `DomainExchange` using the accumulated
   `header.settings.timeout_s` (default `2.0` if header not yet seen — header is always first per format spec, but be
   defensive). Unknown record types are silently skipped.
6. Stamp `parseMs = performance.now() - t0` at the end before posting result.

**Error handling**: only `gzip decode failure` is a hard error (non-gzip file). Truncation is a warning
(`truncated: true`). Missing summary record is handled in `App.vue` (derive totals from exchange records). Unknown
record types are silently dropped.

### Facet logic (`src/lib/filters.ts`)

Two pure functions, no side effects:

**`matchesFacets(ex: Readonly<DomainExchange>, state: Readonly<FacetState>): boolean`** — returns true if the exchange
passes all three facet constraints (AND across axes). `Readonly<>` on both parameters signals purity.

```
perf match:
  'any'     → true
  'fast'    → !ex.isTimeout && ex.rtt <= state.slowMs
  'slow'    → !ex.isTimeout && ex.rtt > state.slowMs
  'timeout' → ex.isTimeout

passes = perfMatch && corrMatch && retryMatch
```

**`clusterMatchesFacets(incident: Readonly<Incident>, state: Readonly<FacetState>): boolean`** — maps facets onto
cluster-level aggregates:

- `perf === 'timeout'` → `incident.timeoutCount > 0`
- `perf === 'slow'` → `incident.peakRtt > state.slowMs && incident.timeoutCount === 0`
- `perf === 'fast'` → `incident.peakRtt <= state.slowMs && incident.timeoutCount === 0`
- `corr === 'violations'` → `incident.violationTypes.size > 0`
- `retryOnly` → `incident.retryCount > 0`

**Tests**: `filters.test.ts` must cover: perf=any passes all, perf=slow excludes timeouts, perf=timeout excludes slow,
corr=violations excludes clean exchanges, retryOnly excludes single-attempt, all combinations AND correctly.

### OID name map (`src/lib/oidNames.ts`)

A static module containing a bundled map of ~2k standard RFC OID prefixes to human-readable names, used for tooltip
display and node labels.

**Sources**: compile the array by hand from public MIB documentation (SNMPv2-MIB, IF-MIB, IP-MIB, TCP-MIB, UDP-MIB,
HOST-RESOURCES-MIB, SNMP-FRAMEWORK-MIB, common Cisco/Juniper/Net-SNMP enterprise prefixes). Ship it as a static
TypeScript `const` literal in `oidNames.ts`. Depth: OID prefixes up to the object level (not table rows). This is a
one-time effort, not auto-generated — there is no MIB compiler in scope. Start with the ~50 most common prefixes from
the MIBs above; expand to ~2k over time. The `lookupOidName` contract and the `satisfies` typing apply regardless of how
many entries exist.

**Data structure**: an array of `[prefix: string, name: string]` pairs, sorted by descending prefix length. This
ordering makes the linear scan implement longest-prefix-first without any special logic.

**`lookupOidName(oid: OidString): string | null`** — iterate the sorted array, return the first `name` where
`oid === prefix || oid.startsWith(prefix + '.')`. Return `null` if no match.

The static array is typed with `satisfies ReadonlyArray<readonly [string, string]>` so the compiler verifies the shape
of each entry at definition time while preserving the exact literal types for each prefix. This prevents accidental
`[name, prefix]` transposition.

**Tests**: cover exact match, prefix match, longest-prefix wins over shorter prefix, and no-match returns null.

### Incident Stack algorithm (`src/lib/incidentStack.ts`)

**`buildIncidents(exchanges: readonly DomainExchange[], slowMs: number): Incident[]`** — pure, deterministic.
Returns incidents sorted by score descending. `ReadonlyArray` signals no mutation of the input array or its elements.

**OID region lookup** (internal, not exported): map from the 8-prefix region list in the spec. Longest-prefix-first
matching (iterate from most-specific to least-specific). Unknown OIDs: `oid.split('.').slice(0, 8).join('.')`. This list
is intentionally separate from the ~2k name map and the OID Tree's short well-known names.

Region prefixes (order matters — longer prefixes before shorter ones sharing the same root):

```
1.3.6.1.2.1.2.2.1  → ifTable
1.3.6.1.2.1.1.    → system
1.3.6.1.2.1.2.    → interfaces
1.3.6.1.2.1.4.    → ip
1.3.6.1.2.1.6.    → tcp
1.3.6.1.2.1.11.   → snmp
1.3.6.1.2.1.25.   → hrSystem
1.3.6.1.4.1.      → enterprises
```

**Anomaly detection**: an exchange is anomalous if
`rtt > slowMs || violations.length > 0 || attemptCount > 1 || isTimeout`. Build a parallel boolean array over all
exchanges for O(n) membership.

**Gap-window clustering** (`GAP_WINDOW = 8`): scan exchanges left to right. When an anomalous exchange is found, open a
new cluster. Extend the cluster when the next anomalous exchange either: (a) has at most `GAP_WINDOW` non-anomalous
exchanges between it and the last cluster member (gap = `indexB − indexA − 1`), or (b) both exchanges share the same OID
region. Non-anomalous exchanges are never cluster members. When no anomalous exchange can extend the cluster, close it.

**Cluster aggregation per cluster**: derive `peakRtt`, `retryCount` (sum of `attemptCount - 1`), `timeoutCount`,
`violationTypes` (Set of all distinct violation strings), `region` (the region name with the highest member count),
`score`.

**Score formula** (from prototype, binding). Use named constants:
```
TIMEOUT_WEIGHT    = 100
VIOL_TYPE_WEIGHT  = 50
RETRY_WEIGHT      = 10
LOG_RTT_WEIGHT    = 5
MEMBER_WEIGHT     = 0.1

score = TIMEOUT_WEIGHT × timeoutCount + VIOL_TYPE_WEIGHT × violationTypes.size
      + RETRY_WEIGHT × retryCount + Math.log10(Math.max(peakRtt, 1)) × LOG_RTT_WEIGHT
      + members.length × MEMBER_WEIGHT
```

**Tests**: cover empty input, single anomalous exchange, gap-window merge, region-based merge regardless of gap, no
non-anomalous exchanges as members, score ordering. Trap: verify a gap of exactly `GAP_WINDOW` (8) merges, and
`GAP_WINDOW + 1` (9) does not.

### OID trie (`src/lib/oidTrie.ts`)

**`buildTrie(exchanges: readonly DomainExchange[], slowMs: number): TrieNode`** — pure, returns a
virtual root node. Receives an already-filtered exchange list (filtered by `matchesFacets` in App.vue); inserts all of them. Each exchange contributes one leaf per unique OID in `ex.responseOids` (which are already 7-arc-truncated in
the domain model). A leaf is `shared = true` if `ex.responseOids.length > 1`.

**Trie insertion**: for each responseOid string, split on `.` to get arc array, walk the trie creating `TrieNode`
children as needed, attach the exchange as a `TrieLeaf` at the deepest node.

**`rollup(node: TrieNode, slowMs: number): void`** — post-order traversal. For each node: aggregate stats from children
(sum `count`, max `maxRtt`, sum `violationCount`) and from own leaves. Severity:
`severity = max(child.severity, leaf_severity)` where leaf severity is `Severity.Violation` if
`exchange.violations.length > 0`, else `Severity.Slow` if `exchange.rtt > slowMs`, else `Severity.Ok`. Takes `slowMs`
directly so the caller can pass `state.slowMs` without coupling `rollup` to `FilterState`.

**`flatten(root: TrieNode): FlatRow[]`** — DFS pre-order. Skip the virtual root itself; start with root's children. A
node row is always emitted; its children and leaves follow only if `node.expanded === true`. Leaf rows are appended
after all child node rows when expanded.

**Auto-expand**: nodes containing anomalous leaves (severity > 0) are expanded by default. Implement as
`autoExpand(node: TrieNode): boolean` — returns true if any descendant has severity > 0, sets `node.expanded = true` if
so. Call before first flatten.

**Well-known names** (OID Tree display, intentionally distinct from other lists):

```
1.3.6.1.2.1.1   → system
1.3.6.1.2.1.2   → interfaces
1.3.6.1.2.1.4   → ip
1.3.6.1.2.1.6   → tcp
1.3.6.1.2.1.11  → snmp
1.3.6.1.2.1.25  → host
1.3.6.1.4.1.9   → cisco
1.3.6.1.6.3.16  → snmpVacm
```

These are looked up at trie insertion time (by exact match on `node.fullOid`) and stored in `TrieNode.name`. The
`lookupOidName()` from `oidNames.ts` serves a different purpose (tooltips); this short list is embedded in `oidTrie.ts`.

**Tests**: cover empty input, single-OID exchange, multi-OID exchange produces multiple leaves with `shared=true`,
rollup aggregates correctly, severity propagates to root, auto-expand sets only nodes with anomalous descendants. Trap:
do not roll stats up from the virtual root's non-leaf data; the root is a synthetic node with arc `''`.

### Minimap + Detail (`MinimapDetail.vue`)

Two canvas elements. `ResizeObserver` on the minimap wrapper drives both canvas width updates; canvas height is fixed
for minimap (`80px`) and fills the remainder for detail.

**Minimap canvas**: covers the full walk duration. One pixel column = one time bucket (pixel count = canvas width). For
each filtered exchange that is anomalous: `col = Math.floor(sentAtMs / maxT * miniW)`. Bucket stores
`{ count, maxRtt, viol, timeout, retry }`. Max bar height = `MINI_H − 4`; bar height is proportional to
`count / maxBucketCount`. Colour priority (per bucket): timeout → `#ef4444`, violation → `#f59e0b`, retry → `#93c5fd`,
slow → `#3b82f6`. `maxT` is the maximum `receivedAtMs` across all exchanges.

After drawing bars, draw the selection rectangle: `rgba(59,130,246,0.22)` fill + `#3b82f6` stroke. Call
`buildBuckets()` + `drawMini()` on any filter change; `buildBuckets()` must re-read the canvas width.

**Detail canvas**: exchanges whose time window overlaps `[winS, winE]` (i.e., `sentAtMs < winE && receivedAtMs > winS`).
One horizontal bar per exchange, ordered by `sentAtMs`. Bar X = `ML + (sentAtMs - winS) / mpp`, bar width =
`max(2, rtt / mpp)`. Retry attempts rendered as stacked bars (use individual `attempt.sent_at` and per-attempt RTT).
Violation marker `!` after the bar. Seq number label on the left (right-aligned). Time-axis ticks at the top; tick
interval chosen from a predefined set (`[0.1, 0.2, 0.5, 1, 2, 5, ...]` seconds) so ticks are at least 60px apart.

`ML = 64` (left margin for seq labels). `MT = 24` (top margin for tick labels). Per-row: `RH = 9` (bar height), `RG = 3`
(gap between rows), `BH = 6` (bar thickness within RH).

**Hover tooltips**: minimap hover shows `{ time offset, event count, max RTT, violation flag }` in a positioned `<div>`.
Detail canvas hover shows `{ seq, % into trace, violation/status, RTT, sent-at time, OID }` in the global tooltip
`<div>`.

**Window interaction**: track `dragging` state and drag start coordinates. A click (no drag) centres the window at the
clicked time with width = `5% × maxT`. Drag on empty area creates a new window. Drag inside the selection rectangle
pans. Drag on the edges (within 4px) resizes. `ArrowLeft`/`ArrowRight` keyboard events (when minimap is focused) shift
by 20% of span.

**Initial window**: `winS = 0`, `winE = maxT × 0.05`. `onData()` (called once after data loads) runs `autoFocus()`: find
the densest/worst bucket (highest `(viol ? 1e9 : 0) + maxRtt × 1000 + count` score) and centre the window there.

**Toolbar**: window range label (e.g., `0.00s – 12.40s (42)`), "Zoom to problems" button (focus the densest problem
bucket, same scoring), "Reset window" button (restore 5% width centred at current midpoint).

Canvas `role="application"`, `aria-label="Timeline minimap"` / `aria-label="Exchange detail"`. Arrow key handling only
fires when the canvas wrapper has focus and active view is minimap.

## Vue 3 traps (read before implementing any component)

These are Vue 3 + Vite + TypeScript-specific pitfalls not obvious from the spec. Apply them throughout the component
tasks.

**Canvas and DOM refs**: obtain canvas element handles with `const canvasRef = ref<HTMLCanvasElement | null>(null)`
bound to `<canvas ref="canvasRef">` in the template. The ref is `null` until mounted — all `getContext()` and
`ResizeObserver` calls must live in `onMounted`, not in the `<script setup>` body.

**Non-reactive trie**: the `TrieNode` tree returned by `buildTrie()` is a plain (non-reactive) object. Do **not** wrap
it in `reactive()` or `ref()` — doing so on a 100k-leaf tree would impose deep reactivity tracking overhead and break
the plan's "mutate `expanded` in place + call re-flatten" strategy. The `computed(() => flatten(oidRoot.value))` in
`App.vue` re-runs because `oidRoot` itself is replaced on filter change; `expanded` mutations are handled by the
explicit re-flatten callback, not by Vue's reactivity.

**`defineProps` / `defineEmits`**: every `.vue` component receiving complex props (`Incident[]`, `FlatRow[]`, callbacks)
must use the typed `defineProps<{ … }>()` generic form. Use `defineEmits<{ … }>()` for events rather than callback props
— Vue's `v-on` / `@event` convention is the idiom. The plan describes "callbacks for close and navigate" etc.; implement
these as emits.

**`v-for` keys**: virtualised rows implemented with absolute positioning and manual DOM insertion (as in the prototype)
work without `v-for` — but if any section uses `v-for` to render visible rows, every item needs a stable `:key` (use
`seq` for exchange rows, `fullOid` for trie node rows).

**`responseOids` source vs prototype**: the prototype (lines 321-325) reads varbinds from `attempts[].response.varbinds`
(per-attempt) as a historical artefact. The schema places `response` at the exchange level, not the attempt level. The
plan correctly uses `exchange.response?.varbinds` — do not copy the prototype's per-attempt path.
