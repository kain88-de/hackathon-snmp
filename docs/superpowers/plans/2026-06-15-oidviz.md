# OIDviz Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **BEFORE ANYTHING ELSE:** Read `docs/dev-guidelines/web-guardrails.md`. It is a hard constraint list — wrong package manager,
> wrong linter, or wrong tsc invocation is a task failure. No exceptions.

**Goal:** Build the OIDviz browser app — a Vue 3 + TypeScript viewer for `.oidtrace.jsonl.gz` files that lets a
monitoring admin answer "is this device behaving correctly?" with three interactive views.

**Architecture:** A single-page app with a two-column layout (fixed 220px sidebar + flex main). All trace parsing
happens off the main thread in a Web Worker. Pure computation modules (filter logic, incident clustering, OID trie) are
plain TypeScript with no Vue dependencies, so they are testable with `bun test` without a browser. Vue 3 Composition API
drives reactivity in components.

**Tech Stack:** Vue 3 · TypeScript · Bun · Vite · `@vitejs/plugin-vue` · Biome (formatting + linting, all rule groups
enabled) · Oxlint (all rules enabled via `-D all`, with `.oxlintrc.json` to suppress false positives) ·
ESLint + `eslint-plugin-vue` (Vue template conventions, `.vue` files only, `--max-warnings 0`) · vue-tsc (template-aware
type checking) · axe-core CLI (rendered a11y gate) · see `docs/dev-guidelines/web-guardrails.md` for the full toolchain
and feedback loop.

### Generated types (`src/lib/types.gen.ts`)

Run once at setup and again after schema changes:

```sh
bunx json-schema-to-typescript ../docs/trace-format.schema.json -o src/lib/types.gen.ts
```

This produces TypeScript interfaces for the five top-level record types: `Header`, `SystemInfo`, `Exchange`, `Event`,
`Summary`. Do not hand-write these.

**Trap — nested type names are not guaranteed.** `Attempt`, `Request`, `Response`, `Settings`, and `Varbind` are inline
object schemas inside `exchange` and `header`, not top-level `$defs`. `json-schema-to-typescript` synthesises names for
them (often something like `Exchange1`, `Header1`, or just inlines them). **Run `just gen-types` in task 1 and inspect
the actual output before writing `model.ts` or `ParseResult`.** The plan uses `Attempt`/`Request`/`Settings` etc. as
readable shorthand; pin the real generated names before implementing.

**`received_at` nullability**: The generated type for the attempt's `received_at` field may be `number | null`. Verify
the generated output preserves this — it must be nullable so timeout detection works without type casts downstream.

### Type contracts (binding — declare before implementing)

Every type below is a contract. Implementers work to these shapes; they do not discover them mid-task. The discipline mirrors the OIDtrace plan: structural vocabularies are enums, structured identifiers are branded types, and module boundaries carry exact signatures. Where the prose elsewhere in this plan disagrees with a shape here, **this section wins**.

The generated names in `types.gen.ts` are confirmed real top-level exports — `Header`, `SystemInfo`, `Exchange`, `Event`, `Summary`, `Varbind`, and `Oid` (a `type Oid = string` alias). The synthesised-name trap from the "Generated types" note did **not** materialise; import these names directly. `Oid` from `types.gen.ts` is plain `string` (schema-level); re-brand at the parse boundary (see `OidString` below).

#### Type-driven development

**Principle: encode correctness in the type system. If a constraint can be a compile error, it must be.** Apply this throughout — to domain quantities (branded types), to state machines (discriminated unions with exhaustiveness), to external data (runtime validation before it enters the typed domain), and to mutation discipline (`readonly` inputs to pure functions).

Specific binding constraints for this codebase:
- `enum`, not `const enum` — Biome bans `const enum` as bundler-incompatible
- `T | null` for present-but-empty values; bare `?` only for genuinely-may-be-omitted keys (`exactOptionalPropertyTypes`)
- No `any` in any public surface

#### Contracts by module

**`model.ts`** — no logic except `asOid`. Only `interface`, `type`, `enum`, and that one constructor.

```ts
export type OidString = string & { readonly _brand: 'OidString' };
export function asOid(s: string): OidString; // sole sanctioned cast — worker only

export interface DomainExchange {
  seq: number;
  rtt: number;               // ms
  isTimeout: boolean;
  violations: string[];      // open enum per format spec — string[] is correct
  attemptCount: number;
  requestOid: OidString;
  responseOids: OidString[]; // unique, 7-arc-truncated; [] when no response
  sentAtMs: number;
  receivedAtMs: number;
}

export interface ParseResult {
  exchanges: DomainExchange[];
  header: Header;
  summary: Summary | null;       // null not absent — exactOptionalPropertyTypes
  systemInfo: SystemInfo | null;
  parseMs: number;
  truncated: boolean;
}

// Facet axes: Performance is mutually exclusive (exactly one state per exchange);
// Correctness is independent; retryOnly is an orthogonal attribute.
export interface FacetState {
  perf: 'any' | 'fast' | 'slow' | 'timeout'; // default 'any'
  corr: 'any' | 'violations';                // default 'any'
  retryOnly: boolean;                         // default false
  slowMs: number;                             // default 1000ms; UI input is seconds
}

export interface Incident {
  startIdx: number;
  endIdx: number;
  startSeq: number;
  endSeq: number;
  members: number[];           // indices into the exchanges array
  peakRtt: number;             // ms
  retryCount: number;          // sum of (attemptCount - 1) over members
  timeoutCount: number;
  violationTypes: Set<string>; // Set encodes "distinct" — score uses .size
  region: string;              // a label, NOT OidString
  score: number;
}

// Retire Severity enum. Replace severity: Severity with flags: TrieNodeFlags.
// Math.max(node.severity, …) as Severity casts are the exact pattern this
// section exists to prevent — they bypass the type system to coerce a number
// back to an enum. flags needs no casts.
export interface TrieNodeFlags {
  slow: boolean;
  violation: boolean;
  retry: boolean; // timeout absent by construction: no response OIDs
}

export interface TrieNode {
  arc: string;               // arc label — plain string, not OidString
  fullOid: OidString;
  name: string | null;
  children: Map<string, TrieNode>;
  leaves: TrieLeaf[];
  expanded: boolean;         // intentionally mutable — only mutable domain field
  stats: { count: number; maxRtt: number; violationCount: number };
  flags: TrieNodeFlags;      // OR-rolled from children + own leaves
}

// oid is the response OID this placement was filed under — differs from
// exchange.requestOid in GETNEXT/BULK walks (inconsistency #12 fix).
export interface TrieLeaf {
  exchange: DomainExchange;
  oid: OidString;  // placement path (response OID), not requestOid
  shared: boolean;
}

export type FlatRow =
  | { kind: 'node'; depth: number; node: TrieNode }
  | { kind: 'leaf'; depth: number; exchange: DomainExchange; oid: OidString; shared: boolean };

export interface WorkerRequest { type: 'parse'; buffer: ArrayBuffer }
export type WorkerResponse =
  | { type: 'result'; data: ParseResult }
  | { type: 'error'; message: string };

export type AppState =
  | { phase: 'landing' }
  | { phase: 'loading' }
  | { phase: 'viewer'; result: ParseResult }
  | { phase: 'error'; message: string };

// 'findings'='Findings', 'incidents'='Incident Stack', 'minimap'='Minimap + Detail', 'oidtree'='OID Tree'
export type ActiveView = 'findings' | 'incidents' | 'minimap' | 'oidtree';
```

### Domain model (`src/lib/model.ts`)

Shapes are in §Type contracts. Behavioral rules that are not obvious from the types:

- **RTT**: `(last.received_at - first.sent_at) * 1000`. If last attempt timed out, `(last.sent_at + timeout_s - first.sent_at) * 1000`. `timeout_s` from `header.settings.timeout_s`, default `2.0` — resolved at parse time in the worker.
- **`responseOids` truncation**: first 7 arcs, rejoin with `.`, deduplicated. `"1.3.6.1.2.1.2.2.1.1.1"` → `"1.3.6.1.2.1.2"`. Empty when no response.
- **`isTimeout`**: last attempt `received_at === null` (not any attempt).
- **Missing summary**: if `result.summary === null`, App.vue derives `oids_seen` from unique `requestOid` count, `end_reason` as `'unknown'`, `violation_counts` from exchanges.

### Parser worker (`src/lib/parser.worker.ts`)

Instantiate with `new Worker(new URL('./lib/parser.worker.ts', import.meta.url), { type: 'module' })` — the `type: 'module'` is required for Vite to bundle it. Transfer `ArrayBuffer` in the postMessage transfer list — do not copy.

Parse algorithm: `DecompressionStream('gzip')` → `TextDecoder(stream: true)` → split on `\n` → `JSON.parse` → dispatch by `rec.type`. Collect `header` (first), `summary` (last wins), `system_info` with `point: 'start'`, enrich all `exchange` records. Default `timeout_s = 2.0` if header not yet seen. Truncation = leftover bytes after reader exhausted → `truncated: true`. Only gzip failure is a hard error; unknown record types silently dropped.

---

### Facet logic (`src/lib/filters.ts`)

Signatures in §Type contracts. Perf match: `'any'` → true, `'fast'` → `!isTimeout && rtt ≤ slowMs`, `'slow'` → `!isTimeout && rtt > slowMs`, `'timeout'` → `isTimeout`. All three axes AND-ed. `clusterMatchesFacets` maps to cluster aggregates: perf=timeout → `timeoutCount > 0`, perf=slow → `peakRtt > slowMs && timeoutCount === 0`.

Tests: perf=any passes all; perf=slow excludes timeouts; perf=timeout excludes non-timeouts; corr=violations excludes clean; retryOnly excludes single-attempt; default (all-any) passes everything.

---

### OID name map (`src/lib/oidNames.ts`)

`[prefix: string, name: string][]` sorted descending by prefix length (longest-prefix-first without extra logic). `lookupOidName(oid: OidString): string | null` — first match where `oid === prefix || oid.startsWith(prefix + '.')`. Typed with `satisfies ReadonlyArray<readonly [string, string]>` to prevent `[name, prefix]` transposition. Start with ~50 common prefixes; the contract holds regardless of count.

Tests: exact match, prefix match, longest-prefix wins, no-match returns null.

---

### Incident Stack algorithm (`src/lib/incidentStack.ts`)

Anomaly: `rtt > slowMs || violations.length > 0 || attemptCount > 1 || isTimeout`. Gap-window clustering (`GAP_WINDOW = 8`): scan left-to-right, merge if gap ≤ 8 **or** same OID region (regardless of gap). Non-anomalous exchanges are never members. Region prefixes (longer before shorter sharing the same root) per spec. Score uses named constants:

```
TIMEOUT_WEIGHT=100, VIOL_TYPE_WEIGHT=50, RETRY_WEIGHT=10, LOG_RTT_WEIGHT=5, MEMBER_WEIGHT=0.1
score = 100×timeoutCount + 50×violationTypes.size + 10×retryCount + log10(max(peakRtt,1))×5 + members.length×0.1
```

Trap: gap of exactly 8 merges; gap of 9 does not. `retryCount` = sum of `attemptCount - 1`, not count of exchanges with retries.

---

### OID trie (`src/lib/oidTrie.ts`)

Receives a pre-filtered list (App.vue calls `matchesFacets` first — `buildTrie` does not filter). Each exchange contributes one `TrieLeaf` per `responseOid` in `ex.responseOids`; `leaf.oid` is set to the specific placement OID. `shared = true` when `responseOids.length > 1`.

`rollup` — post-order. Flags: `slow ||= child.flags.slow || (ex.rtt > slowMs)`, same for `violation` and `retry`. No `Math.max(… ) as Severity` casts. `autoExpand` condition: `flags.slow || flags.violation || flags.retry`. Well-known names (system/interfaces/ip/tcp/snmp/host/cisco/snmpVacm) embedded in `oidTrie.ts`, looked up at insertion by exact `fullOid` match.

Traps: skip the synthetic root (`arc = ''`) in `flatten`; `rollup` visits children before own leaves; `autoExpand` must recurse into deep descendants.

Shapes and props/emits are in §Type contracts. The spec is the authoritative source for structure and UX. Only traps below.

**Sidebar**: radio groups need `<fieldset>`/`<legend>` for a11y (not just labels). Fixture buttons `fetch('/../traceformat/examples/{name}.oidtrace.jsonl.gz')` then post the buffer — they do not open a file picker. The `/../` prefix works because `vite.config.ts` sets `server.fs.allow: ['..']`.

**IncidentStack**: chip colours use `--dim-*` tokens (`--dim-timeout`, `--dim-violation`, `--dim-slow`, `--dim-retry`), **not** the retired `severity-err/warn/info` classes. Modal exchange table shows only `matchesFacets`-passing members.

**MinimapDetail**: canvas reads `--dim-*` colours via `getComputedStyle` at draw time (dark mode). Layout constants: `MINI_H=80`, `ML=64`, `MT=24`, `RH=9`, `RG=3`, `BH=6`, `MAX_H=32_767`. Detail canvas height: `MT + rows × (RH + RG) + 20` capped at `MAX_H`. `autoFocus()` on data load: densest/worst bucket scored `(viol ? 1e9 : 0) + maxRtt × 1000 + count`.

**OidTree**: `TrieNode.expanded` is mutated in place; the component emits `reflatten` and the parent re-flattens. After collapse-all, the flat list shrinks — the virtualiser must not hold stale row indices. Leaf rows display `row.oid` (the placement response OID), **not** `exchange.requestOid`.

## Vue 3 traps

- **Canvas refs**: `ref<HTMLCanvasElement | null>(null)` — the ref is `null` until mounted. All `getContext()` and `ResizeObserver` calls must be in `onMounted`.
- **Non-reactive trie**: do **not** wrap `TrieNode` in `reactive()` or `ref()` — deep reactivity on 100k leaves kills performance. `oidRoot` is replaced on facet change (triggers `computed`); `expanded` mutations are handled by the explicit reflatten emit.
- **`defineEmits`** not callback props — Vue's `@event` convention throughout.
- **`v-for` keys**: `seq` for exchange rows, `fullOid` for trie nodes.
- **`responseOids` source**: use `exchange.response?.varbinds`, **not** `attempts[].response.varbinds` — the prototype has a historical artefact on the per-attempt path.
