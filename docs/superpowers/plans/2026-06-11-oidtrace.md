## Architecture (binding)

Two workspace packages: `traceformat/` (shared format types, own tests) and
`oidtrace/` (source under `src/oidtrace/`, tests under `tests/`).

### Type-driven rules

**Strict by default.** A bare `str`/`int` may cross a module boundary only when the
domain is genuinely open: wire tags, device-returned ids, and the three format fields
deliberately left open for evolution (`violations`, `event.kind`, `end_reason` — open
for _readers_; the producer side uses the closed `StrEnum`s in `traceformat.vocab`).
Everything else is closed: structural vocabularies are enums in the schema itself
(snmp version, system_info point, pdu — codegen emits them as Enums); identifiers with
structure get value types (`Oid`).

**Domain types vs format models.** The codebase has deliberate type pairs with exactly
one conversion site each:

- `oidtrace.oid.Oid` (behavioral value class: ordering, `in_subtree`) vs generated
  `models.Oid` (wire-string form) — converted at the **records boundary** (`str(oid)`),
  same place `codec.Varbind` becomes the varbind dict.
- `transport.Attempt` (frozen dataclass, `error: AttemptError` enum, I/O domain) vs
  generated `models.Attempt` (record form, `error: str`) — converted in the **walker**.

I/O and behavior types stay enum-rich and method-bearing; format models stay
schema-derived. Never let a generated model leak below the records/walker boundary.

### `oid.py` — OID value type

- `@dataclass(frozen=True, order=True, slots=True) class Oid` over `arcs: tuple[int, ...]`
- `Oid.from_str("1.3.6.1")` — ValueError on garbage (ASCII digits only, no leading zeros)
- `str(oid)` → dotted form; readable `__repr__`; `oid.in_subtree(root) -> bool`
- `order=True` over arcs gives correct OID ordering (bisect-compatible); frozen gives hashing

### `traceformat` package (workspace member) — shared format types

Standalone docs: spec `docs/superpowers/specs/2026-06-12-traceformat-design.md`, plan
`docs/superpowers/plans/2026-06-12-traceformat.md`. Summary for readers of this plan:
schema-generated pydantic models (never hand-edited; `types-fresh` drift gate),
`TraceRecord` union, `dump_record`/`parse_record` with exclude-unset semantics, and the
producer vocabulary StrEnums (`Violation`, `EndReason`, `EventKind`, `AttemptError`)
referenced throughout the contracts below.

### `ber.py` — minimal BER primitives

- `tlv(tag, payload)`, `encode_int(v, tag=0x02)`, `encode_oid(Oid)`
- `read_tlv(buf, i) -> (tag, payload, next_i)` — ValueError on any truncation/overrun
- `decode_int(body, *, signed=True)`, `decode_oid(body) -> Oid`
- Tags stay `int` deliberately: the wire is an open set.

### `codec.py` — SNMP v2c messages

- `encode_getbulk(request_id, oid, non_repeaters, max_repetitions, community=b"public") -> bytes`
- `encode_response(request_id, varbinds: Sequence[(Oid, tag, value_bytes)], ...) -> bytes` (emulator side)
- `decode_message(raw) -> Message | Malformed` — tolerant; catches ValueError only
- `Varbind(oid, tag, value)` frozen, derived `vtype` (per format § 5, unknown → `tag:0xNN`) and `vlen`
- `Message(pdu_tag, request_id, f1, f2, varbinds: tuple, raw)` — `f1`/`f2` generic: one decoder serves responses (error_status/index) and emulator-received requests (non-repeaters/max-repetitions)
- `Malformed(raw, error)`; `_TAG_NAMES` (private — consumed only via `Varbind.vtype`); `EXCEPTION_TAGS = {0x80, 0x81, 0x82}`

### `violations.py` — pure checks

- `check_exchange(*, sent_id, returned_id, prev_oid, varbinds, response_raw, strays) -> list[Violation]`
- Produces `REQUEST_ID_MISMATCH`, `OID_NOT_INCREASING` (equal counts; exception tags skipped), `DUPLICATE_RESPONSE`. `MALFORMED_BER` is added by the walker.

### `records.py` — record builders (pydantic models)

- Builders return `traceformat` model instances (typed, validated at construction);
  no `Any` anywhere in the signatures:
  - `header_record(*, tool, started_at, label, session_id, run, runs_total, snmp_version, settings: models.Settings) -> Header`
  - `exchange_record(*, seq, request: models.Request, attempts: Sequence[models.Attempt], response_request_id: int | None, response_error_status: int | None, response_error_index: int | None, varbinds: Sequence[codec.Varbind], strays: Sequence[models.StrayResponse], violations: Sequence[Violation], malformed: models.Malformed | None) -> Exchange`
  - `event_record(*, at, kind: EventKind, detail=None) -> Event`
  - `summary_record(*, at, exchanges, oids_seen, end_reason: EndReason, violation_counts: Mapping[Violation, int]) -> Summary`
  - `system_info_record` (format completeness; v1 walker never calls it)
- Conditional keys stay absent via unset fields; producer-closed enums only (no `| str`)
- Varbind values are unreachable from any built record
- Builder outputs still schema-validated in tests (belt and braces against codegen gaps)

### `tracefile.py` — gzip-JSONL I/O

- `TraceWriter(path)`: `write(record: TraceRecord)` appends one JSON line + flush (per-record durability); context manager
- `read_trace(path) -> Iterator[TraceRecord]`: complete records only, validated via pydantic TypeAdapter (round-trip guarantee); stops quietly at truncation; raises on records that are valid JSON but violate the format (our own bug, not tolerance territory)

### `transport.py` — the only I/O

- `Attempt(sent_at, received_at, error: AttemptError | None = None)` frozen, built once with final outcome — deliberately distinct from generated `models.Attempt` (see Type-driven rules); the walker converts
- `ExchangeIO(attempts: tuple, response: (received_at, raw) | None, strays: tuple[(received_at, raw), ...])` frozen
- `Transport` = `typing.Protocol` with `async exchange(raw, *, timeout_s, retries) -> ExchangeIO` — the walker depends on the protocol; a FakeTransport replaying scripted ExchangeIO values is the walker's unit-test fixture
- `UdpTransport.create(host, port, rel)` classmethod (asyncio datagram endpoint, connected socket); **async context manager** (no bare try/close at call sites)
- `exchange` sends, waits, retries with byte-identical resend; never validates; ICMP via `error_received` → `AttemptError`; after a response, drain immediately-available datagrams into strays. **Timestamps are stamped in the protocol callbacks (event time), never at dequeue** — stray arrival gaps must stay honest (format § 4.3 rule 3)

### `walker.py` — the orchestrator

- `WalkSettings(bulk_size: int = 10 (>= 1), timeout_s=2.0, retries=2, start_oid: Oid = Oid.from_str("1.3.6.1"), time_budget_s=None, give_up_after=3, community=b"public")` frozen
- `run_walk(host, port, *, settings, path, label=None, session_id=None, run=1, runs_total=1, sinks: Sequence[RecordSink] = ()) -> EndReason` where `RecordSink = Callable[[TraceRecord], None]`
- The walk core is an **async generator**: `walk_records(transport, *, rel, settings, label, session_id, run, runs_total) -> AsyncIterator[TraceRecord]` — yields header → exchanges → events → summary; consumers decide per record and own cancellation (breaking out of `async for` just ends iteration; the generator must NOT catch CancelledError to yield more — illegal after GeneratorExit). Partial consumers MUST close deterministically (`contextlib.aclosing`/`aclose()`), never rely on GC. Cancellation rules: rely on 3.11+ `asyncio.timeout`/uncancel (external cancel ≠ timeout) and 3.13+ TaskGroup precedence; the adapter re-raises CancelledError after emitting the interrupted summary, sync sinks only during cleanup (verified against the 3.14 release notes: no semantic changes).
- `WalkStats` accumulator (`observe(record)`; exposes exchanges/oids_seen/violation_counts): the stream is self-sufficient, so any consumer can synthesize an INTERRUPTED summary from the records it consumed. As built: the generator keeps its own in-subtree `oids_seen` accounting for the normal summary; `WalkStats`-from-records may exceed it by up to bulk_size on the terminal exchange (documented in its docstring) — acceptable for interrupted summaries.
- `walk_with_transport` remains as the **sink-pumping adapter**: pumps the generator into `sinks: Sequence[RecordSink]`, catches CancelledError → emits the interrupted summary from its own WalkStats → re-raises; returns the EndReason from the final summary. Takes the required shared `rel` clock and never touches a file — a `TraceWriter.write` bound method is simply the canonical sink. `run_walk` is the file-producing composition (opens TraceWriter(path) as CM, prepends it to caller sinks, manages the transport CM). A future HTML/SSE host drives the core with its own sink list, file optional.
- Sink failure contract (v1): sinks are trusted and may crash the walk (the writer is the product; test sinks are trivial). When a non-trivial sink lands (SSE), wrap caller sinks so they cannot abort capture — revisit then.
- Owns one shared `rel()` clock passed to both transport and loop; all times `round(rel, 6)`; every record through one `emit()` that fans out to all sinks; no Optional callback, empty tuple is the null object
- Owns the domain→format conversions: `transport.Attempt` → `models.Attempt`, building `models.Request`/`models.StrayResponse`/`models.Malformed` for the record builders
- GetBulk-only loop; termination: endOfMibView → COMPLETED, left subtree → COMPLETED, oid-not-increasing → OID_LOOP (+ event), time budget → TIME_BUDGET_EXCEEDED (+ event), give_up_after consecutive response-less exchanges → UNRESPONSIVE; CancelledError → summary INTERRUPTED, re-raise

### `cli.py` — argparse entry

- `main(argv) -> int`; subcommand `walk(host, --port, --out, --label, --bulk-size, --timeout, --retries, --start-oid, --time-budget)`
- DNS resolved up front and `--start-oid` parsed to `Oid` — operator errors exit 2 with stderr message, no trace file
- Community from `OIDTRACE_COMMUNITY` env (default `public`); live progress on stderr via an appended sink; terminal summary + trace path on stdout; Ctrl-C falls through to the same summary + path (the INTERRUPTED summary is already flushed) and exits 0

### Logging

Stdlib `logging`, module loggers (`logging.getLogger(__name__)`) — handlers ARE the
adjustable sinks; hosts attach what they want. Rules: **libraries emit, never
configure** (only `cli.main` calls the setup); the pure layers (oid/ber/codec/records)
stay log-free; log lines obey trace privacy (never values, never the community
string). Emission map: walker INFO start (settings)/end reason/events, DEBUG
per-exchange decisions; transport DEBUG send/timeout/retry/ICMP/stray; cli INFO
resolved target + trace path. CLI: `-v/--verbose` counting flag — default WARNING,
`-v` INFO, `-vv` DEBUG — stderr handler; the `\r` progress line shows only at default
verbosity (logs supersede it under `-v`).

**Binding data-structure decisions:** OIDs are a frozen dataclass over
`arcs: tuple[int, ...]` (string comparison mis-orders `1.3.6.1.10` vs `1.3.6.1.2`;
tuple-based ordering compares correctly — test this). Strings only at boundaries via
`Oid.from_str` / `str(oid)`. Tolerant decode is a union return, not exceptions. Records are dicts from
builders, schema-validated in tests. Varbind value bytes exist in memory only.

**Everything crossing a boundary is a frozen dataclass with immutable fields:**
`Oid`, `Varbind`, `Message` (`varbinds: tuple[Varbind, ...]`), `Malformed`, `Attempt`
(constructed once with its final outcome — the transport never mutates an attempt
mid-retry), `ExchangeIO` (tuple fields — FakeTransport fixtures are safely shareable
across tests), `WalkSettings`, `Quirks`, `EmuDevice`. Records are pydantic models
generated from the schema (validated at construction, serialized via `dump_record`'s
exclude-unset semantics). Mutable state
exists in exactly two places: the walker's loop state and the transport's internal
event queue.

**Runtime data flow (one I/O point, everything else pure):** per exchange, the walker
— the only orchestrator, strictly sequential, sole owner of state (cursor, seq, t0,
violation counts) — runs:

```
codec.encode_getbulk(...)          pure: values -> bytes
await transport.exchange(...)      THE ONLY I/O: bytes -> ExchangeIO
codec.decode_message(...)          pure: bytes -> Message | Malformed
violations.check_exchange(...)     pure: values -> [str]
records.exchange_record(...)       pure: values -> dict
emit(dict) -> TraceWriter + on_record   dumb sinks; nothing flows back
termination decision + cursor advance   pure, from decoded varbinds
```

Knowledge boundaries: transport treats packets as opaque bytes (no SNMP — that is why
it cannot accidentally validate); codec knows SNMP but never sees a socket; violations,
records are value→value; tracefile knows JSON+gzip, not SNMP. Import DAG:
`oid ← ber ← codec ← {violations, walker, emulator}`; records/tracefile/transport are
independent legs; walker is the join; cli sits on top. The emulator mirrors the same
layering server-side (decode → behavior source → encode), sharing only the codec.

Errors become data at each boundary going up: ber raises ValueError → codec catches
into `Malformed` → transport maps OS errors to `Attempt.error` strings → walker turns
everything into records plus an `end_reason` → cli maps that to exit codes. Device
misbehavior never propagates as an exception ("misbehavior is data", structurally).

**Where we test:** the trace file is the primary observable — walker tests assert on
written records, never internals. Wire bytes are the second contract — codec output
must decode via pysnmp **spec-driven** (`decoder.decode(raw, asn1Spec=Message())`;
pyasn1's schemaless decoder cannot walk SNMP PDUs — PoC lesson).
